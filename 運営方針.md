# Zenn 運営方針

## 目的

XRPL を中心にブロックチェーン・Web3 の技術記事を Zenn で継続的に公開する。

---

## 記事のファイル名

形式: **`{chain}-{category1}-{category2}-{date}.md`**

| 要素 | 説明 | 例 |
|------|------|-----|
| chain | チェーン名 | `xrpl` |
| category1 | 難易度 or ジャンル | `beginner` / `advanced` / `defi` / `news` |
| category2 | トピック（省略可） | `ledger` / `token` / `nft` |
| date | 公開日 | `2026-02-10` |

例: `xrpl-beginner-basics-2026-02-10.md` / `xrpl-defi-token-2026-02-10.md`

---

## スクリプトと実行順

### 1. トレンド調査 `trends-survey.sh`

- **実行**: 毎日
- **出力**: 月別ファイル `trends/YYYY-MM.md`
- **内容**: ブロックチェーン市場の「人気カテゴリランキング」
- **挙動**: ファイルがなければ新規作成、あれば上書き更新

```bash
./trends-survey.sh
```

### 2. 記事作成・公開 `zenn-auto.sh`

- **実行**: 記事を出すタイミングで
- **出力**: `articles/` に記事を追加し、GitHub に push
- **前提**: 必要に応じて先に `trends-survey.sh` を実行（トレンドを参照する場合はその月の `trends/YYYY-MM.md` が存在すること）

```bash
./zenn-auto.sh
```

---

## 記事のルール

- **言語**: 日本語
- **対象**: XRPL を中心に、初心者でも読める技術記事
- **frontmatter**: title / topics は内容に合わせて自動生成。`published` は初期値 `false`（公開時に `true` に変更）
- **公開**: スクリプト実行で `git add` → `commit` → `push` まで実施

---

## ディレクトリ構成

```
zenn/
├── articles/     # Zenn 記事（ファイル名は上記形式）
├── trends/       # 月別トレンド（YYYY-MM.md）
├── trends-survey.sh
├── zenn-auto.sh
└── 運営方針.md   # 本ドキュメント
```
