## はじめに

「XRPLって名前は聞いたことあるけど、結局なに？」という方に向けて、XRP Ledger（XRPL）の仕組みと特徴をできるだけわかりやすくまとめました。

- 開発者視点での始め方

## XRPLとは

**XRP Ledger（XRPL）** は、2012年にリリースされたオープンソースの分散型台帳（ブロックチェーン）です。Ripple社の共同創業者である David Schwartz、Jed McCaleb、Arthur Britto らによって開発されました。

ネイティブ通貨として **XRP** を持ち、主に **高速・低コストな決済** を目的として設計されています。

### ポイント

| 項目 | 内容 |
| --- | --- |
| リリース | 2012年 |
| ネイティブ通貨 | XRP |
| トランザクション確定時間 | 約3〜5秒 |
| 手数料 | 約0.00001 XRP（ほぼゼロ） |
| コンセンサス方式 | XRP Ledger Consensus Protocol |
| マイニング | 不要 |

## 他のブロックチェーンとの違い

### Bitcoinとの比較

Bitcoinは **Proof of Work（PoW）** を採用しており、マイニングによってブロックが生成されます。そのため電力消費が大きく、トランザクションの確定にも10分程度かかります。

一方XRPLは、マイニングが不要で **数秒で決済が完了** します。環境負荷の面でも大きなアドバンテージがあります。

### Ethereumとの比較

Ethereumは汎用的なスマートコントラクトプラットフォームとして設計されていますが、ガス代が高くなりがちです。

XRPLは決済に特化しており、手数料が極めて低いのが特徴です。ただし、Ethereumのような自由度の高いスマートコントラクトは標準では提供していません（後述するHooksやサイドチェーンで拡張可能）。

### 比較表

| 特徴 | Bitcoin | Ethereum | XRPL |
| --- | --- | --- | --- |
| 確定時間 | 約10分 | 約12秒 | 約3〜5秒 |
| 手数料 | 高い | 変動（高くなりがち） | ほぼゼロ |
| コンセンサス | PoW | PoS | 独自プロトコル |
| スマートコントラクト | 限定的 | 汎用 | 限定的（拡張中） |
| 環境負荷 | 大きい | 小さい（PoS移行後） | 非常に小さい |

## コンセンサスアルゴリズム

XRPLが採用している **XRP Ledger Consensus Protocol** について解説します。

### 概要

XRPLでは、**バリデータ（検証ノード）** と呼ばれるサーバー群がトランザクションの正当性を検証します。各バリデータは **UNL（Unique Node List）** と呼ばれる信頼するノードのリストを持っており、UNL内のノードの **80%以上** が合意した場合にトランザクションが確定します。

### 流れ

1. ユーザーがトランザクションをネットワークに送信
2. バリデータがトランザクションを受信し、検証を開始
3. 各バリデータが「このトランザクションを承認するか」を提案
4. UNL内のバリデータ間で複数ラウンドの投票が行われる
5. 80%以上の合意が得られたら、レジャー（台帳）に記録
6. 新しいレジャーが閉じられ、確定（約3〜5秒）

### PoWやPoSとの違い

- **マイニング不要** → 電力消費が少ない
- **ステーキング不要** → 参入障壁が低い
- **確定が速い** → フォーク（分岐）が原理的に起きにくい

## XRPLの主な機能

XRPLは決済だけでなく、さまざまな機能をプロトコルレベルで備えています。

### 1. 分散型取引所（DEX）

XRPLには **ネイティブのDEX** が組み込まれています。オーダーブック方式で、誰でもトークンの売買注文を出すことができます。

```
// オファー作成の例（概念）
{
  "TransactionType": "OfferCreate",
  "TakerPays": { "currency": "USD", "issuer": "rIssuer...", "value": "100" },
  "TakerGets": "50000000"  // 50 XRP（drops単位）
}
```

### 2. トークン発行（IOUs）

XRPL上では、誰でも独自のトークンを発行できます。これは **Trust Line** という仕組みで管理されます。

- 法定通貨のトークン化（USD、JPYなど）
- ポイントやクーポンの発行
- ステーブルコインの作成

### 3. NFT（XLS-20）

**XLS-20** という規格により、XRPL上でもNFTの発行・売買が可能です。

- ミント（発行）
- バーン（焼却）
- ブローカー経由の売買
- ロイヤリティの設定

### 4. AMM（自動マーケットメーカー）

2024年に導入された **AMM機能** により、流動性プールを使ったトークン交換が可能になりました。従来のオーダーブック型DEXと組み合わせることで、より効率的な取引が実現します。

### 5. Hooks（スマートコントラクトの拡張）

**Hooks** は、XRPLのトランザクションにカスタムロジックを追加できる機能です。WebAssembly（Wasm）ベースで動作し、軽量なスマートコントラクトのような役割を果たします。

## 開発者向け：XRPLを触ってみよう

### テストネットを使う

XRPLには **Testnet** と **Devnet** が用意されており、無料でXRPを取得して開発を試せます。

テストネットのFaucet：
https://faucet.altnet.rippletest.net/accounts

### xrpl.jsを使った基本例

JavaScriptでXRPLに接続する最小限のコードです。

```javascript
const xrpl = require("xrpl");

async function main() {
  // テストネットに接続
  const client = new xrpl.Client("wss://s.altnet.rippletest.net:51233");
  await client.connect();

  // テスト用ウォレットを生成
  const wallet = xrpl.Wallet.generate();
  console.log("アドレス:", wallet.classicAddress);
  console.log("シークレット:", wallet.seed);

  // 残高を確認（Faucetで入金後）
  // const response = await client.request({
  //   command: "account_info",
  //   account: wallet.classicAddress,
  // });
  // console.log("残高:", response.result.account_data.Balance);

  await client.disconnect();
}

main();
```

## まとめ

XRPLの特徴を振り返ります。

- **高速**：トランザクションが3〜5秒で確定
- **低コスト**：手数料がほぼゼロ
- **省エネ**：マイニング不要で環境に優しい
- **多機能**：DEX、トークン発行、NFT、AMMなどがネイティブで使える
- **開発しやすい**：テストネットとSDKが充実

ブロックチェーンを初めて触る方にとっても、XRPLは手数料の心配なく気軽に試せる環境が整っています。まずはテストネットで遊んでみるところから始めてみてください。
